# 뷰 (VIEW)

* 하나 이상의 테이블을 합쳐 만든 <U>가상의 테이블 </U>
    * 데이터를 실제 물리적인 디스크에 저장하지 않음
* `SELECT`문을 통한 결과를 가상의 테이블로 정의하여 살제 테이블 처럼 사용할 수 있음 
 
<br></br> 

 ## 특징
**1. 논리적 독립성을 제공**
  
**2. 데이터의 접근 제어 기능을 제공**

**3. 데이터 관리 단순화 시킬 수 있음**

**4. 재사용성 상승**



## 한계
**1. 물리적 테이블 갱신 작업의 제한**

**2. 테이블 변경에 영향 받음**

**3. 인덱스를 직접 가질 수 없음**


<br></br> 

## 뷰 문법

### 생성
```sql
CREATE VIEW 뷰이름 
AS SELECT
FROM 
WHERE 
..
```

### 조회
```sql
SELECT * FROM 뷰이름;
```

### 수정
```sql
CREATE OR REPLACE VIEW 뷰이름
AS SELECT
...
```

### 삭제
```sql
DROP VIEW 뷰이름;
```
