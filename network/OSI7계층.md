# OSI 7 Layer
* 국제 표준화 기구(ISO)에서 개발된 네트워크의 한 모델
    * 네트워크 프로토콜을 나타내는 표준 규약
* 프로토콜 디자인과 통신 계층을 나누어 개념적으로 나타낸 모델
* 네트워크 과정이 7개의 계층으로 모듈화 되어 있음
    - 각 계층은 특정한 역할과 기능을 수행
    - 각 계층은 다른 계층과 독립적으로 동작
    - 각 계층은 인접한 계층과의 통신하여서 다른 계층에 미치는 영향을 최소화함

## 1. 응용 계층 (Application Layer)

* 네트워크 통신의 최상위 계층으로, 사용자가 사용하는 `애플리케이션 소프트웨어(브라우저, 메일)`와 가장 가까운 계층
* 실제 사용자가 입력한 데이터를 하위 계층으로 전달하거나, 하위 계층에서 올라온 데이터를 사용자에게 보여줌
* 개발자가 네트워크 기능을 쉽게 사용할 수 있도록, 애플리케이션에 네트워크 서비스(API, 프로토콜)를 제공함
* 대표적인 프로토콜:  
  - **HTTP / HTTPS**: 웹 브라우징  
  - **SMTP**: 이메일 발신  
  - **FTP**: 파일 전송  
  - **Telnet / SSH**: 원격 접속  
  - **DNS**: 도메인 이름 → IP 주소 변환

## 2. 표현 계층 (Presentation Layer)

* 응용 계층에서 내려온 <U>데이터를 전송하기 적합한 형식으로 변환(인코딩)하고, 수신한 데이터를 응용 계층이 이해할 수 있도록 복원(디코딩)</U>하는 계층
* 데이터의 포맷, 문자 인코딩, 압축, 암호화/복호화 등의 처리를 담당
    * 송신 측: 데이터를 변환, 압축, 암호화  
    * 수신 측: 데이터를 복원, 해제, 복호화
    - **문자 인코딩**: ASCII, UTF-8
    - **압축**: ZIP, MPEG
    - **이미지/미디어 포맷**: JPEG, MP3
    - **암호화**: SSL, TLS (※ 전송 계층과 표현 계층의 경계에서 동작)

## 3. 세션 계층 (Session Layer)

* 통신을 하는 두 시스템(또는 프로세스) 사이에 `세션(session)`이라는 논리적 연결을 생성하고,**세션의 시작/유지/종료**를 관리하는 계층
    * `세션(session)`  
        - 하나의 대화가 시작되고 끝날 때까지의 통신 구간  
        - 예: 로그인 → 데이터 교환 → 로그아웃까지
* 주요 기능
  - 세션 연결 설정, 동기화, 종료
  - 통신 시 흐름 제어, 충돌 방지, 재전송 관리
  - 연결 유지 타이밍, 회복 등

* 프로토콜
  - NetBIOS: 세션 기능 일부 담당
  - RPC (Remote Procedure Call)
  - 일부 SSL/TLS의 핸드셰이크 기능

## 4. 전송 계층 (Transport Layer)

* 통신하는 양 끝단(클라이언트 ↔ 서버)의 **프로세스 간 데이터 전송을 담당**
* 데이터를 여러 조각(세그먼트)으로 나누고, 목적지에서 재조립할 수 있게 함
* 오류 발생 시 재전송, 순서 보장, 흐름 제어, 혼잡 제어 등 **신뢰성 있는 데이터 전송** 기능을 제공함

* 프로토콜
  - **TCP (Transmission Control Protocol)**  
    → 연결지향 / 신뢰성 보장 / 순서 보장 / 재전송 지원
  - **UDP (User Datagram Protocol)**  
    → 비연결지향 / 신뢰성 없음 / 빠르고 단순함 / 실시간 스트리밍 등에 사용

* 데이터 단위:
  - TCP: **세그먼트 (Segment)**
  - UDP: **데이터그램 (Datagram)**

* 이 계층에서 **포트 번호**를 사용하여, 여러 앱들이 동시에 통신할 수 있도록 도와줌  
  → ex) 포트 80 (HTTP), 포트 443 (HTTPS)
 
## 5. 네트워크 계층 (Network Layer)
* **데이터가 목적지까지 어떤 경로(라우팅)로 전달되어야 하는지 결정**하는 계층
* 송신자와 수신자의 IP 주소를 이용해 **논리적 주소 지정과 경로 선택**을 수행
* 데이터 단위: **패킷(Packet)**  
* 주요 기능
  - 라우팅 (Routing)
  - 논리 주소 지정 (IP 주소)
  - 패킷 전달 및 포워딩
  - 네트워크 간 연결 

* 프로토콜
  - **IP (Internet Protocol)** – 주소 지정과 라우팅
  - **ICMP (Internet Control Message Protocol)** – 오류 메시지 전달 (ex: ping)
  - **IGMP (Internet Group Management Protocol)** – 멀티캐스트 그룹 관리
  - **(보조적으로) ARP, RARP** – IP ↔ MAC 주소 매핑

## 6. 데이터 링크 계층 (Data Link Layer)

* 같은 네트워크 내에 있는 장치들 간의 **물리적 전송을 신뢰성 있게 수행**
* 네트워크 계층에서 받은 **패킷을 프레임(Frame)** 단위로 감싸고,
  **MAC 주소를 이용해** 목적지까지 전달
* **오류 검출**, 흐름 제어, 충돌 방지 등 기능을 담당

* 하위 2개 서브 계층으로 나뉨
  - **MAC (Media Access Control)**  
    → 물리 주소(MAC 주소) 지정, 채널 접근 제어
  - **LLC (Logical Link Control)**  
    → 에러 검출, 프레임 동기화 등

* 프로토콜
  - **Ethernet (이더넷)**
  - **Wi-Fi (802.11)**
  - **PPP (Point to Point Protocol)**
  - **Frame Relay**

* 데이터 단위: **프레임 (frame)**


## 7. 물리 계층 (Physical Layer)

* 데이터를 **0과 1의 비트**로 표현하여, **전기/광/무선 신호**로 변환해 전송
* 상위 계층에서 받은 **프레임을 물리적으로 전달**하는 역할
* **오류 검출이나 흐름 제어는 하지 않음** – 단순히 "보내는 역할"만 수행
* 사용하는 전송 매체에 따라 **전기, 광, 무선** 등 다양한 신호 방식이 있음

* 주요 구성 요소:
  - 케이블 (UTP, 광케이블 등)
  - 리피터, 허브
  - 커넥터, 전송 회선
  - 전기 신호 / 광 신호 / 무선 신호 등

* 데이터 단위: **비트 (bit)**
