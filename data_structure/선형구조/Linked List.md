# 연결 리스트 (Linked List)

![alt text](<../설명사진/[자료구조-선형구조] linkedlist01.png>)
* **Linked List란?** 데이터들이 메모리 상에 연속적으로 붙어 있지 않고, <U>서로의 주소를 가리키는 선으로 연결되어 일렬로 늘어선 형태</U>의 선형 자료구조
* **node:** Linked List의 각 연결 데이터 덩어리
    * `data` 와 `next pointer` 두 가지의 요소로 이루어짐
* 리스트의 첫 번째의 노드를 `머리(head)`이라고 하고 리스트의 마지막 노드를 `null(참조를 가지지 않음)`이라고 부름

## 종류

### 단방향 연결 리스트 (singly linked list)
![alt text](<../설명사진/[자료구조-선형구조] linkedlist02.png>)
* `다음 노드의 주소`와, `데이터`를 가지는 일반적인 연결 리스트의 형태 
* 포인터를 하나만 사용하므로 메모리 공간을 굉장히 알뜰하게 사용함
* 이전의 데이터를 탐색하고 싶으면 다시 `헤드(head)`부터 재탐색을 해야함

```java
class Node {
    Node next; // 다음 노드 주소를 저장하는 필드
    int data;
};
```

### 양방향 연결 리스트 (doubly linked list)
![alt text](<../설명사진/[자료구조-선형구조] linkedlist03.png>)
* `다음 노드의 주소`와, `이전 노드의 주소`와 `데이터`를 가지는 연결 리스트의 형태 
* 양방향 탐색이 가능하다
* 포인터를 두개나 가져야 하므로 메모리 공간을 더 소모하게 됨
```java
class Node {
    Node next; // 다음 노드 주소를 저장하는 필드
    Node prev; // 이전 노드 주소를 저장하는 필드
    int data; 
};
```

# 메모리(RAM) 관점에서의 Linked List
![alt text](<../설명사진/[자료구조-선형구조] linkedlist01.png>)
* **Linked List**는 <U>메모리의 빈공간을 미리 정해둘 필요가 없고</U> 빈 공간에 자유롭게 들어가는 구조를 가짐
* **동적 메모리 할당:** 데이터가 추가될 때마다 메모리의 남는 공간에 새로운 노드를 하나 만들고, 이전 노드에 그 주소만 알려주면(선으로 연결하면) 끝남

# 시간 복잡도

## 읽기 or 접근 = O(N)
* index 번호로 한번에 가지 않음 
* 반드시 `맨 앞(head)`부터 선을 타고 하나 씩 탐색함

## 검색 = O(N)
* 특정 값이 어디 있는지 찾으려면 처음부터 끝까지 모든 노드를 일일이 열어봐야 함

## 데이터 삽입, 수정, 삭제 (모든 위치) = O(1)
* 노드를 이어주는 화살표의 방향만 바꿔주면 됨
* 문제는 제어하기 위해 탐색을 해야하는데 탐색하러 가는 과정이 0(N)이라는 것..
